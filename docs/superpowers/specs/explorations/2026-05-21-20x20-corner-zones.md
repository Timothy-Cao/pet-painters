# Exploration: 20×20 Board with 5×5 Corner Home Zones

**Status**: Deferred — decision support only, no implementation.

## The Idea

Replace the current 16×16 board with a 20×20 board (400 tiles). Each player's home zone becomes a 5×5 square in opposite corners rather than 3 contiguous rows at top/bottom. Pets face diagonally toward the center of the board. Optional follow-on: introduce 3×3 pets (e.g. a redesigned Dragon) that fit more naturally on the larger field.

## Geometric Implications

| Metric | Current 16×16 | Proposed 20×20 |
|---|---|---|
| Total tiles | 256 | 400 |
| Win threshold (75%) | 192 tiles | 300 tiles |
| Home zone size | 48 tiles (3 rows) | 25 tiles (5×5 corner) |
| Home-to-center distance | ~8 tiles (straight) | ~13 tiles (diagonal) |

Pets travel roughly 60% further before engaging. Player-vs-player contact happens in the center quadrant, approached diagonally. Pets currently facing N/S would need a new "march toward opponent corner" brain mode or diagonal-facing support (NE/SW/NW/SE).

## Impact on Existing Balance Work

Six rounds of tier-list iteration are effectively invalidated. Every cost, stat, and win-threshold value was tuned for 256 tiles and a ~8-tile engagement range. On a 400-tile board with 13-tile approach paths:

- Energy budget of 20 is too low for a 300-tile target — needs scaling (estimate: 30–35).
- All cost/stat numbers need re-baseline from scratch.
- 1×1 pets occupy 1/400 of the board vs 1/256 today — relatively smaller, less impactful per tile.
- Sprint painters (Mouse, Eagle, Spider) still reach the front fast but then grind a longer midfield.

## Estimated Implementation Cost

| File | Change |
|---|---|
| `src/sim/constants.ts` | `BOARD_SIZE`, `HOME_ROWS` → `HOME_CORNER_SIZE`, win threshold |
| `src/sim/deploy.ts` | Home-zone generation: rows → corner squares; anchor validation |
| `src/render/board.ts` | Coordinate math for 20×20 canvas scaling |
| `src/render/deploy-preview.ts` | Deployment UX preview coordinates |
| `src/sim/behaviors.ts` | `walkOrTurnAtWall`, `facingDelta` — add diagonal facing if needed |
| `src/sim/pets/*.ts` | Any pet whose behavior hardcodes board geometry |
| `src/sim/__tests__/*.ts` | All tests referencing board size or home-zone positions |

Rough estimate: **25–40 hours** for a clean implementation + re-baseline balance sweep.

## Expected Meta Changes

- **Longer matches**: more distance = more time before engagement; match length likely 2–3× current.
- **Big pets gain**: Whale and a 3×3 Dragon have more room to establish territory before contact.
- **Sustain painters gain**: Bear, Cat, Elephant benefit from painting more tiles during the longer approach.
- **Sprint painters lose ground**: Mouse, Eagle, Spider reach the front fast but face a prolonged midfield grind.
- **Diagonal engagement**: pets facing N/S walk a longer path to reach an opponent in an opposite corner; diagonal facing may become necessary for competitive play.

## Risks

- Match lengths above ~5 minutes become a game-session chore; no playtest signal yet on whether longer is better.
- Every energy, cost, and win-threshold value needs re-tuning from zero.
- Corner home zones break `deploy.ts` assumptions (currently contiguous rows); push-chain geometry at board edges changes.
- A 3×3 pet has 3 front tiles and a 9-tile footprint — push chains are more complex; movement correctness needs verification against edge cases.

## Recommendation

**Defer to v2.** Six rounds of tuning on the current 16×16 represent real calibration data. Discarding that without a clear playtest signal ("corner geometry is more fun" or "matches feel too short") is premature. The better next step is to test the 3×3 pet hypothesis on the existing board — which is exactly what the Dragon 3×3 experiment (separate task) does.

## Alternative: Smaller Step

Keep the 16×16 board but reshape the home zone from 3 contiguous rows to a **4×4 corner square**. This tests whether corner geometry (diagonal engagement, smaller home) is the interesting design change, independent of board size. Implementation cost is much lower (~5 hours), and it preserves most existing balance tuning.
